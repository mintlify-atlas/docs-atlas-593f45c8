```bash Response
> Text splitting is handled by the TextSplitter class in src/rag/text_splitter.py. It uses LangChain's RecursiveCharacterTextSplitter with language-aware splitting. The class maps file extensions to specific Language types (supporting 25+ languages like Python, JavaScript, TypeScript, Java, C++, Go, Rust, etc.). For each document, it selects the appropriate splitter based on file extension, using language-specific separators. Default chunk size is 1000 characters with 200 character overlap. Files without recognized extensions use generic splitting with newline and space separators.
```

```bash Response
> The RAGRetriever class in src/rag/rag_retriever.py handles document retrieval from the vector store. It takes a query string, generates an embedding using the EmbeddingManager, performs a cosine similarity search in ChromaDB, and returns the top-K most relevant document chunks. ## Data flow and process queries
```

### Understanding the pipeline

<CodeGroup>
```bash Query
Ask anything ('exit' to quit): How are documents stored in the vector database?
```

```bash Response
Running RAG for query: How are documents stored in the vector database?
Retrieving documents for query: 'How are documents stored in the vector database?'
Top K: 5, Score threshold: 0.0
Retrieved 5 documents (after filtering)

> In src/main.py:41-44, documents are stored in ChromaDB through the VectorStore class. The process:
> 1. Extract page_content from each document chunk
> 2. Generate embeddings for all texts using EmbeddingManager
> 3. Call vector_store.add_documents() with both chunks and their embeddings
> 4. Store in a collection named after the repo (with '/' replaced by '_')
> 5. Persist to disk at ~/.RepoRAGX/vector_store/
>
> ChromaDB uses cosine similarity indexing for efficient retrieval.
```
</CodeGroup>

```bash Response
> When you ask a question in src/main.py:49-54:
> 1. Your query is passed to llm.rag()
> 2. The GroqLLM generates an embedding for your query
> 3. RAGRetriever performs cosine similarity search in ChromaDB
> 4. Top 5 most relevant code chunks are retrieved with file paths
> 5. Context is formatted with file headers and content
> 6. A prompt is constructed with context + your question
> 7. The Groq LLM (llama-3.3-70b-versatile) generates an answer
> 8. The answer is printed to the console
> 9. The prompt reappears for your next question
```
